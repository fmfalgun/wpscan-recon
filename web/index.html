> WHAT IS WPSCAN RECON
WordPress powers ~43% of the web. wpscan-recon runs a full WordPress fingerprint: WordPress version + outdated check, installed plugins with versions and CVEs (if API token), theme detection, user enumeration via REST API or author scan, and exposure checks (xmlrpc.php — system.multicall brute-force risk; readme.html — version disclosure). Structured JSON output saved per-scan.
> WHY THIS SCRIPT
Raw wpscan output is interactive text — hard to parse, no cache, no history. wpscan-recon adds 24h SQLite caching (no re-scanning same target within a day), structured JSON output, HTTP fallback mode (works without wpscan installed via urllib), and community submission.

> USAGE
# Basic scan
python3 wpscan-recon.py --url https://target.com

# Save JSON output to file
python3 wpscan-recon.py --url https://target.com --output results.json

# Enable CVE lookup (WPScan API token required)
python3 wpscan-recon.py --url https://target.com --api-token YOUR_TOKEN

# Enumerate: vp=vulnerable plugins, vt=vulnerable themes, u=users, ap=all plugins
python3 wpscan-recon.py --url https://target.com --enumerate vp,vt,u,ap

# Bypass cache and force fresh scan
python3 wpscan-recon.py --url https://target.com --no-cache

# Submit results to the WordPress Board
python3 wpscan-recon.py --url https://target.com --submit

# Reconfigure API token or submission settings
python3 wpscan-recon.py --reconfigure

> XMLRPC RISK
xmlrpc.php exposes system.multicall — allows 500+ password attempts in a single HTTP request, bypassing rate limiting. Standard target for credential stuffing and brute-force attacks against WordPress admin accounts.
  • Single request = hundreds of login attempts
  • Bypasses fail2ban and rate-limit plugins
  • Also exploitable for pingback SSRF (CVE-2013-0235 pattern)
  • Fix: disable via mu-plugin or .htaccess deny rule
> README EXPOSURE
/readme.html reveals the exact WordPress version in plaintext. Attackers use this to target known CVEs without triggering any active scanning. A passive, zero-noise fingerprint vector.
  • Reveals exact WP version — narrows CVE search space
  • No authentication, no scanning — public GET request
  • Fix: delete readme.html or block via .htaccess
  • Also check: /license.txt, /wp-admin/install.php
